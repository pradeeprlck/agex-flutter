# AgEx Home Screen Widget - Setup Guide

This directory contains the native Android files for the AgEx voice assistant home screen widget.
Since the Flutter project's `android/` directory is generated by `flutter create`, these files
need to be copied into the correct locations after the project is initialized.

## Files Overview

| File | Copy To |
|------|---------|
| `AgexWidgetProvider.kt` | `android/app/src/main/kotlin/<your_package>/AgexWidgetProvider.kt` |
| `agex_widget.xml` | `android/app/src/main/res/layout/agex_widget.xml` |
| `agex_widget_info.xml` | `android/app/src/main/res/xml/agex_widget_info.xml` |
| `widget_background.xml` | `android/app/src/main/res/drawable/widget_background.xml` |
| `mic_button_bg.xml` | `android/app/src/main/res/drawable/mic_button_bg.xml` |
| `chip_background.xml` | `android/app/src/main/res/drawable/chip_background.xml` |

## Step-by-Step Integration

### 1. Copy Files

After running `flutter create` or when the `android/` directory exists:

```bash
# Assuming package name: com.agex.app
PACKAGE_DIR="android/app/src/main/kotlin/com/agex/app"

# Copy Kotlin provider
cp native/android/widget/AgexWidgetProvider.kt $PACKAGE_DIR/

# Create resource directories if needed
mkdir -p android/app/src/main/res/layout
mkdir -p android/app/src/main/res/drawable
mkdir -p android/app/src/main/res/xml

# Copy resources
cp native/android/widget/agex_widget.xml android/app/src/main/res/layout/
cp native/android/widget/agex_widget_info.xml android/app/src/main/res/xml/
cp native/android/widget/widget_background.xml android/app/src/main/res/drawable/
cp native/android/widget/mic_button_bg.xml android/app/src/main/res/drawable/
cp native/android/widget/chip_background.xml android/app/src/main/res/drawable/
```

### 2. Update Package Name in AgexWidgetProvider.kt

Open `AgexWidgetProvider.kt` and change the package declaration to match your app:

```kotlin
package com.agex.app  // ← change to your actual package name
```

### 3. Update AndroidManifest.xml

Add the following inside the `<application>` tag in `android/app/src/main/AndroidManifest.xml`:

```xml
<!-- Home Screen Widget -->
<receiver
    android:name=".AgexWidgetProvider"
    android:exported="true"
    android:label="AgEx Voice Assistant">
    <intent-filter>
        <action android:name="android.appwidget.action.APPWIDGET_UPDATE" />
    </intent-filter>
    <meta-data
        android:name="android.appwidget.provider"
        android:resource="@xml/agex_widget_info" />
</receiver>
```

Also add this deep link intent-filter inside the existing `<activity>` tag for `MainActivity`:

```xml
<!-- Deep link for voice activation from widget -->
<intent-filter>
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />
    <data android:scheme="agex" android:host="voice" />
</intent-filter>
```

### 4. Add String Resource

In `android/app/src/main/res/values/strings.xml`, add:

```xml
<string name="widget_description">Quick access to AgEx voice assistant, weather, prices, and crop diagnosis</string>
```

### 5. Handle Deep Link in Flutter

The deep link `agex://voice` is already handled in the Flutter app's `VoiceAssistantWidget`.
When the app opens via this deep link, the voice assistant bottom sheet will activate automatically.

To enable deep link handling in GoRouter, ensure the router configuration in `app_router.dart`
includes the redirect logic:

```dart
// In your GoRouter configuration, the VoiceAssistantFab already listens for
// the agex://voice deep link via url_launcher / platform channel.
```

### 6. Add Mic Icon

You'll need a white mic icon for the widget button. Either:

- **Option A**: Use Android's built-in `@android:drawable/ic_btn_speak_now`
- **Option B**: Add a custom `ic_mic_white.png` to `android/app/src/main/res/drawable/`
  (24dp × 24dp, white color, PNG or vector drawable)

The current layout references `@android:drawable/ic_btn_speak_now` which works on most devices.

## Widget Features

- **Tap anywhere on widget** → Opens AgEx app
- **Tap mic button** → Opens app and activates voice assistant (`agex://voice`)
- **Quick action chips** → Weather, Prices, Diagnose (deep links to respective screens)
- **Auto-updates** every hour (configurable in `agex_widget_info.xml`)
- **Resizable** both horizontally and vertically
- **Minimum size**: 250dp × 110dp

## Customization

### Change Widget Colors
- Green theme: Edit `mic_button_bg.xml` (`#2E7D32`)
- Background: Edit `widget_background.xml`
- Chips: Edit `chip_background.xml`

### Change Update Frequency
Edit `updatePeriodMillis` in `agex_widget_info.xml` (minimum 30 minutes = 1800000ms)

### Add More Quick Actions
Edit `agex_widget.xml` to add more chip TextViews in the `chipsContainer` LinearLayout,
and add corresponding click handlers in `AgexWidgetProvider.kt`.
